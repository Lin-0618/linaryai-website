场景地图

自动化不是炫技，是把重复工作收起来

先从低风险、可检查、有明确停止条件的任务开始。

CI

PR 与 CI 盯盘

定时检查失败构建、Review 评论和测试结果，把需要人工确认的点集中汇报。

W

周报与项目摘要

从文档、Issue、提交记录或连接器上下文里整理进展、风险和下一步。

R

发布前检查

按清单检查版本说明、配置项、迁移脚本、截图和文档遗漏。

D

资料定期清理

把零散文件、表格和聊天记录整理成可审阅草稿，避免信息散在各处。

S

安全和权限提醒

提醒 API Key、配置文件、日志和外部连接器的敏感信息处理边界。

T

任务续跑

让一个线程按固定节奏继续检查长任务，而不是靠人记得回来追。

写法建议

一个自动化提示词应该说清五件事

1任务

明确每次醒来要检查什么，不要泛泛写“帮我看看”。

2频率

选择固定间隔、每日、每周或自定义 cron。

3范围

指定项目、仓库、工作树、数据来源和允许读取的内容。

4停止条件

说明什么时候归档、什么时候提醒人、什么时候继续等。

高风险自动化不要直接放开：涉及删除、支付、客户数据、生产配置和外部写操作时，应该保留人工确认。
