FAQ

安装、配置和自动化常见问题

把用户最常问的问题集中在这里，既方便客服，也方便搜索引擎理解页面。

LinaryAI Codex 是 OpenAI 官方产品吗？

不是。LinaryAI Codex 是面向中文用户的配置、教程和便利工具入口。Codex 产品本身、功能可用性和账号权限以 OpenAI 官方页面与用户账号显示为准。

安装助手会上传 API Key 吗？

安装页不收集 API Key。安装器会把 Key 写入本机 Codex 配置，不会把 Key 放进公开日志、命令行参数或网页表单。

Windows 提示“不明发布者”怎么办？

当前安装包未做商业代码签名时，Windows 可能提示 SmartScreen。选择“更多信息”后点击“仍要运行”。正式分发建议购买代码签名证书。

macOS 提示来自不明开发者怎么办？

先运行包内放行脚本，或在系统设置的隐私与安全性里选择“仍要打开”。正式分发建议 Apple Developer 签名和公证。

Codex 没有中文怎么办？

安装器会写入 localeOverride = "zh-CN" 并设置默认中文回答。配置完成后请完全退出 Codex 再重新打开一次。

自动化适合哪些任务？

适合重复、可检查、有明确停止条件的任务，例如 PR/CI 盯盘、发布清单检查、周报草稿、资料整理和定时提醒。

什么时候应该写 Skill？

当一个流程会重复出现、需要固定步骤、参考资料或脚本时，适合沉淀成 Skill。一次性任务用普通提示词即可。

插件、Skills、MCP 有什么区别？

Skills 更像可复用工作流说明；MCP 负责连接外部工具和上下文；插件可以把 Skills、应用连接和 MCP 服务打包成可安装能力。

日志在哪里？

Windows 日志优先在 %LOCALAPPDATA%\LinaryAI\CodexInstaller\logs\LinaryAI-Codex-install.log。macOS 日志在桌面 LinaryAI-Codex-setup.log。发给技术支持前请确认不要包含 API Key。

密钥只在本机处理

使用自动化、插件和外部连接器时，也建议按最小权限原则逐项开启。涉及删除、支付、客户数据、生产配置和外部写操作时，应该保留人工确认。
